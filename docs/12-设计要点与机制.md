# 12 · 设计要点与机制

## 12.1 性能优化

| 优化点 | 实现位置 | 说明 |
| --- | --- | --- |
| **Creep 名字缓存** | `cache.creep.js` | 避免每 tick 多次 `_.filter(Game.creeps)`，O(1) 计数。仅在「初始化/孵化/死亡」刷新 |
| **矿点坐标缓存** | `cache.sources.js` + `state.js` | 房间固定，首 tick 初始化后直接读缓存；缓存失效时 fallback 到 `room.find` 重建 |
| **矿点空位缓存** | `state.sourceSlotCount` | 每 tick 第一次 `getFreeSlotCount` 时计算并缓存，后续命中缓存 |
| **Spawn 距离缓存** | `state.sourceSpawnDist` | Chebyshev 距离，房间固定不变 |
| **Tower ID 缓存** | `state.towerIds` | 命中时用 `Game.getObjectById`（O(1)），失效时全量扫描刷新 |
| **Room.find 缓存** | collector（TTL=20）、transporter（TTL=5） | 按需缓存 sources/containers/dropped/tombstones/structures |
| **路径复用** | `moveTo` 的 `reusePath` | collector=10, transporter=20 |
| **瞬态状态分离** | `state.js` | 不写 `Memory`，减少序列化开销 |
| **调度循环用缓存名单** | `main.js` | 遍历 `state.allNames` 而非 `for...in Game.creeps` |
| **collector 计数用 byRole** | `role.collector.js` | `_countCollectorsAtSource` 遍历 `state.byRole['collector']` 子集而非全部 creep |

## 12.2 死锁/抖动修复

| 问题 | 修复机制 | 实现位置 |
| --- | --- | --- |
| **残余能量死锁** | builder 能量 < 5、upgrader/repairer 能量 < 1 时立即切回采集 | `role.builder/upgrader/repairer.js` 状态切换 |
| **矿点满位顺延** | `harvestNearest` 检测 `hasFreeSlot=false` 自动顺延下一矿点 | `cache.sources.js` |
| **Container↔Spawn 往返抖动** | transporter 未满容量不切送货阶段；无送货目标时保持 carry 阶段不切 empty | `role.transporter.js` `_executePickup` / `_doDelivery` |
| **站在阻挡结构上抖动** | road/rampart 不算阻挡，避免「移开→走回→移开」循环；仅对 Container/Spawn 等非道路结构移开 | `role.transporter.js` `_moveToPickup` / `_moveToDeliver` |
| **采集者卡住自恢复** | 50 ticks 无移动进入 20 ticks 恢复模式，重置状态与锁 | `role.collector.js` `_healthCheck` |
| **transporter 停滞重置** | 50 ticks 无移动强制重置所有状态与锁 | `role.transporter.js` `_healthCheck` |
| **资源锁超时** | 30 ticks 自动释放，防止死 creep 持锁 | `role.transporter.js` `_isPickupLocked` / `_isDeliverLocked` |
| **任务超时回收** | assigned/in_progress 超 300 ticks 自动归档为 TIMED_OUT | `task.scheduler.js` `checkTimeouts` |
| **死 creep 持锁** | 每 tick 检测 sourceLocks/targetLocks/active 任务，持锁 creep 不存活则释放 | `task.scheduler.js` `checkDeadlocks` |
| **moveTo 失败循环** | 连续 5 次移动失败放弃目标；取货/送货错误 3 次放弃 | `role.transporter.js` `_handleMoveError` / `_handlePickupError` / `_handleDeliveryError` |
| **`getAdjacentPosition` 不存在** | `_moveAwayFromTarget` 手动用方向常量计算相邻格 | `role.transporter.js` |

## 12.3 短缺优先机制

`state.creepShortage` 为 true 时（任一角色 `count < roleTargets[role]`）：

| 角色 | 行为 |
| --- | --- |
| builder | 立即暂停建造，调用 `_moveAwayFromSpawn`（远离 Spawn 5 格避免挡搬运者） |
| upgrader | 立即暂停升级，调用 `_moveAwayFromSpawn` |
| collector/transporter | 不受影响，继续工作保证能量链路 |
| manager.spawn | 优先孵化短缺角色（按优先级队列 harvester → collector → ...） |

**目的**：把能量与 Spawn 槽位让给孵化，保证房间规模能恢复。

## 12.4 紧急模式机制

基地毁灭后（creep 总数 ≤ 3）自动进入紧急模式：

| 阶段 | 配置 | 特点 |
| --- | --- | --- |
| 正常模式 | `roleTargets`（harvester=0, collector=6, transporter=5, ...） | 等待高能量孵化更高级 creep |
| 紧急模式 | `emergencyRoleTargets`（harvester=3, collector=2, transporter=1, 其余=0） | **不等待高能量**，立即孵化；以 harvester 为主，不依赖 Container |

**退出条件**：`creepCache.count('harvester') >= 3`，恢复正常配置。

## 12.5 让位协调机制

collector 优先级最高（纯采集是能量链路源头），连续无法到达 source 时可请求其他角色让位：

```
优先级：collector > 其他角色

让位请求触发条件：连续 3 次 ERR_NO_PATH 无法到达同一 source
让位者选择优先级：
  1. transporter/upgrader/builder/repairer（YIELDABLE_ROLES）
  2. 其他 collector/harvester

让位执行：
  - 被请求者 _yieldUntil = Game.time + 5
  - _yieldTarget = source 外圈 3-5 格处的空地
  - run() 入口 checkYield 返回 true 时跳过正常逻辑，moveTo 让位目标
```

## 12.6 资源锁机制

### transporter 锁（`Memory._transporterLocks`）

| 锁类型 | 格式 | 上限 | 超时 | 用途 |
| --- | --- | --- | --- | --- |
| 取货锁 `pickups` | `{id: creepName}` | 单锁 | 30 ticks | 防止多 transporter 抢同一取货点 |
| 送货锁 `delivers` | `{id: [creepName, ...]}` | 3 个 | 30 ticks | 允许多搬运者同时送货到大容量目标 |

### 任务调度锁（`Memory.tasks`）

| 锁类型 | 格式 | 用途 |
| --- | --- | --- |
| `sourceLocks` | `{sourceId: creepName}` | source 资源锁，一个 source 同一时刻只分配给一个 creep |
| `targetLocks` | `{targetId: [creepName, ...]}` | target 资源锁，支持多 creep |

## 12.7 房间变更自恢复

主循环每 tick 检测 `state.spawnRoomName` 与当前 spawn 房间名是否一致：

```
不一致 → _purgeRoomState()
  ├─ 清理 Memory.tasks.active 中 source/target 失效的任务
  ├─ 清理 Memory.tasks.targetLocks 失效锁
  ├─ 清理 Memory._transporterLocks 失效锁
  ├─ 清理 Memory._transporterLockTimestamps
  ├─ state._cacheReady = false  下 tick 重新初始化
  ├─ state.sourceIds/sourceData/sourceSpawnDist/towerIds = []
  └─ 打印日志
```

**目的**：基地毁灭后迁移到新房间时，自动清理旧房间的残留状态，避免失效引用导致错误。

## 12.8 战斗角色预留

`body.config.js` 预留了 `COMBAT_ROLES`（GUARD/RANGER/HEALER/CLAIMER）与 `COMBAT_BODY_TEMPLATES`，当前未启用，为后续 PvP/扩张功能预留接口。

启用需要：
1. 创建对应的 `role.guard.js` 等 role 文件。
2. 在 `config.roleTargets` 添加角色目标数。
3. 在 `main.js` 角色分派 switch 中添加 case。
4. 在 `body.config.js` 的 `getBody` switch 中添加 case（已有模板）。
