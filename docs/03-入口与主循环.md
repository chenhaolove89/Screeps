# 03 · 入口与主循环

本章涵盖入口层与全局配置/状态三个文件：[`main.js`](../main.js)、[`state.js`](../state.js)、[`config.js`](../config.js)。

---

## 3.1 main.js — 入口与主循环

`main.js` 是 Screeps 引擎每 tick 调用的入口，导出 `loop` 函数。它只做编排，不包含业务逻辑。

### 顶部依赖

```js
var roleHarvester   = require('role.harvester');
var roleCollector   = require('role.collector');
var roleTransporter = require('role.transporter');
var roleUpgrader    = require('role.upgrader');
var roleBuilder     = require('role.builder');
var roleRepairer    = require('role.repairer');
var managerSpawn    = require('manager.spawn');
var managerTower    = require('manager.tower');
var taskScheduler   = require('task.scheduler');
var state           = require('state');
var creepCache      = require('cache.creep');
```

### `_purgeRoomState()` — 房间级状态清理

在房间毁灭或迁移到新房间时调用，清理引用已不存在建筑的任务与锁：

1. 遍历 `Memory.tasks.active`，对 `sourceId` 与 `targetId` 都失效的任务标记 `timed_out` 并删除。
2. 清理 `Memory.tasks.targetLocks` 中已不存在的建筑锁。
3. 清理 `Memory._transporterLocks.pickups` / `delivers` 中失效建筑。
4. 清理 `Memory._transporterLockTimestamps` 中失效时间戳。
5. 打印 `[Main] 房间级状态已清理`。

### `module.exports.loop()` — 主循环执行顺序

每 tick 执行：

1. **重置瞬态缓存**：`state.sourceSlotCount = {}`（矿点空位检测每 tick 重算）。
2. **Spawn 动态发现**：遍历 `Game.spawns` 取第一个可用 spawn，写入 `state.spawnName` 与 `state.spawnRoomName`。
3. **房间变更检测**：若 `state.spawnRoomName` 与新 spawn 房间名不同，调用 `_purgeRoomState()` 清理失效状态，重置 `state._cacheReady = false`、`sourceIds/sourceData/sourceSpawnDist/towerIds`，并打印日志。
4. **首 tick 初始化**（`!state._cacheReady`）：
   - 无可用 spawn → 跳过初始化，等下一 tick。
   - `creepCache.build()` 全量构建 Creep 名字缓存。
   - 遍历 spawn 房间内所有 source，初始化 `state.sourceIds` 与 `state.sourceData`（坐标缓存，房间固定只初始化一次）。
5. **任务调度器初始化**：`taskScheduler.init()`。
6. **防御塔**：`managerTower.run()`。
7. **清理死亡 Creep 内存 + 同步缓存**：遍历 `Memory.creeps`，对已不存在的 creep 调用 `creepCache.remove(name)` 并 `delete Memory.creeps[name]`。
8. **任务调度器维护**：`taskScheduler.checkTimeouts()` + `taskScheduler.checkDeadlocks()`。
9. **检测角色短缺**：`state.creepShortage = managerSpawn.checkShortage()`，重置 `state.spawningRole = null`。
10. **角色调度**：遍历 `state.allNames` 缓存名单，按 `creep.memory.role` 分派到对应 role 模块的 `run(creep)`。
11. **自动孵化**：`if (state.spawnName) managerSpawn.run(state.spawnName)`。

> 调度循环使用缓存名单 `state.allNames` 而非 `for...in Game.creeps`，避免每 tick 全量扫描。对刚死亡但缓存未及时清理的 creep 做了 `if (!creep) continue` 兜底。

### 角色分派表

| `creep.memory.role` | 调用 |
| --- | --- |
| `harvester` | `roleHarvester.run(creep)` |
| `collector` | `roleCollector.run(creep)` |
| `transporter` | `roleTransporter.run(creep)` |
| `upgrader` | `roleUpgrader.run(creep)` |
| `builder` | `roleBuilder.run(creep)` |
| `repairer` | `roleRepairer.run(creep)` |

## 3.2 config.js — 全局配置

集中管理全局可调参数，所有阈值统一在此修改。

| 字段 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `roleTargets` | `Object` | `{harvester:0, collector:6, transporter:5, upgrader:6, builder:5, repairer:1}` | 各角色目标数量 |
| `emergencyRoleTargets` | `Object` | `{harvester:3, collector:2, transporter:1, upgrader:0, builder:0, repairer:0}` | 紧急模式角色配置（基地毁灭后重启用，以 harvester 为主不依赖 Container） |
| `emergencyCreepThreshold` | `number` | `3` | 紧急模式判定阈值：存活 creep 总数 ≤ 此值触发 |
| `spawnEnergyThreshold` | `number` | `200` | 孵化能量门槛，低于此值不孵化 |
| `bodyConfig.debug` | `boolean` | `false` | 是否打印 body 部件生成日志 |
| `bodyConfig.forceDefault` | `boolean` | `false` | 强制使用默认 1:1:1 配比（调试用） |
| `bodyConfig.forceTier` | `number|null` | `null` | 强制使用指定能量阶段（null=自动） |

> 当前配置已将 harvester 目标数设为 0，主力采集改为 collector + transporter 分工模式。

## 3.3 state.js — 共享瞬态状态

`state.js` 是一个单例对象，用于在模块之间传递 **tick 内瞬态信息**，**不读写 `Memory`**。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `creepShortage` | `boolean` | 当前是否有角色短缺（builder/upgrader 据此暂停） |
| `spawningRole` | `string|null` | 当前正在孵化的角色 |
| `allNames` | `string[]` | 所有存活 creep 名字列表（调度循环用） |
| `byRole` | `Object<string,string[]>` | 按角色分组的名字列表（快速计数） |
| `_cacheReady` | `boolean` | 缓存是否已初始化 |
| `sourceIds` | `string[]` | 所有矿点 ID（首 tick 初始化） |
| `sourceData` | `Object<string,{x,y,roomName}>` | 矿点坐标缓存 |
| `sourceSlotCount` | `Object<string,number>` | 每 tick 矿点空闲格数量（瞬态） |
| `sourceSpawnDist` | `Object<string,number>` | 矿点到 Spawn 的 Chebyshev 距离缓存 |
| `spawnName` | `string|null` | 当前主 spawn 名称（动态发现，不再硬编码） |
| `spawnRoomName` | `string|null` | 当前主 spawn 所在房间名（检测房间变更用） |
| `towerIds` | `string[]` | Tower ID 缓存（首次 tick 或缓存失效时刷新） |
| `emergencyMode` | `boolean` | 是否处于紧急模式（基地重启中，以 harvester 为主） |

### 设计要点

- **瞬态性**：所有字段每 tick 重算或增量更新，不跨 tick 序列化，减少 `Memory` 序列化开销。
- **唯一例外**：`emergencyMode` 虽在 state 中，但由 `manager.spawn.js` 的 `_checkEmergency()` 每 tick 重新评估，逻辑上等价于瞬态。
- **房间变更安全**：`spawnName`/`spawnRoomName` 为 null 时，主循环会跳过孵化，等待 spawn 重建。
