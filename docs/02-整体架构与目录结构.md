# 02 · 整体架构与目录结构

## 2.1 五层架构

项目采用**单房间、模块化、缓存优先**的架构，自顶向下分为五层：

```
┌─────────────────────────────────────────────────────────────┐
│  入口层    │  main.js  (每 tick 主循环：调度总控)            │
├─────────────────────────────────────────────────────────────┤
│  配置/状态 │  config.js  (角色目标/阈值/紧急模式配置)        │
│            │  state.js   (tick 内瞬态共享状态)               │
├─────────────────────────────────────────────────────────────┤
│  缓存层    │  cache.creep.js   (Creep 名字/角色计数缓存)     │
│            │  cache.sources.js (矿点坐标/距离/空位缓存)      │
├─────────────────────────────────────────────────────────────┤
│  管理器层  │  manager.spawn.js (孵化管理 + 紧急模式)         │
│            │  manager.tower.js (防御塔管理)                  │
│            │  task.scheduler.js(跨角色任务调度框架)          │
│            │  body.config.js   (Creep 身体部件策略)          │
├─────────────────────────────────────────────────────────────┤
│  角色层    │  role.harvester.js   role.collector.js          │
│            │  role.transporter.js role.upgrader.js           │
│            │  role.builder.js     role.repairer.js           │
└─────────────────────────────────────────────────────────────┘
```

### 架构特点

1. **主循环极简**：`main.js` 只做编排，业务逻辑下沉到管理器与角色模块。
2. **瞬态状态与持久状态分离**：`state.js` 只在 tick 内有效，不写 `Memory`；持久状态由各模块按需写入 `Memory.creeps` / `Memory.tasks` / `Memory._transporterLocks`。
3. **缓存驱动**：角色调度、短缺检测、矿点选择均走缓存，O(1) 计数。
4. **职责单一**：每个 role 文件只负责一种 Creep 行为；manager 负责横切关注点（孵化、防御、任务调度）。

## 2.2 目录结构

```
default/
├── main.js                  # 入口：每 tick 主循环
├── config.js                # 全局配置（角色目标、阈值、紧急模式）
├── state.js                 # tick 内瞬态共享状态
├── body.config.js           # Creep 身体部件策略
├── cache.creep.js           # Creep 名字缓存
├── cache.sources.js         # 矿点缓存
├── manager.spawn.js         # 孵化管理器
├── manager.tower.js         # 防御塔管理器
├── task.scheduler.js        # 任务调度框架
├── role.harvester.js        # 角色：采集+运输（兼容旧逻辑）
├── role.collector.js        # 角色：纯采集
├── role.transporter.js      # 角色：搬运
├── role.upgrader.js         # 角色：升级控制器
├── role.builder.js          # 角色：建造
├── role.repairer.js         # 角色：修理
├── code_wiki.md             # 旧版单文件 wiki（保留）
├── docs/                    # 本文档目录
└── .gitignore
```

## 2.3 文件职责表

| 文件 | 类型 | 职责 |
| --- | --- | --- |
| [`main.js`](../main.js) | 入口 | 每 tick 主循环：重置瞬态缓存 → Spawn 动态发现 → 首 tick 初始化 → 防御塔 → 清理死亡 Creep → 检测短缺 → 调度角色 → 自动孵化 |
| [`config.js`](../config.js) | 配置 | 各角色目标数量、紧急模式角色配置、孵化能量阈值、body 调试开关 |
| [`state.js`](../state.js) | 状态 | tick 内共享瞬态状态（短缺标记、缓存名单、矿点数据、Spawn/房间信息、紧急模式标记） |
| [`body.config.js`](../body.config.js) | 配置 | 按角色与能量阶段生成差异化 body 部件组合 |
| [`cache.creep.js`](../cache.creep.js) | 缓存 | Creep 名字缓存：`build/add/remove/count` |
| [`cache.sources.js`](../cache.sources.js) | 缓存 | 矿点缓存：距离排序、空闲格检测、满位顺延采集 |
| [`manager.spawn.js`](../manager.spawn.js) | 管理器 | 按优先级检测缺口、动态生成 body、执行孵化；含紧急模式检测 |
| [`manager.tower.js`](../manager.tower.js) | 管理器 | 防御塔：攻击敌人 > 紧急维修普通建筑 > 维修 Wall/Rampart |
| [`task.scheduler.js`](../task.scheduler.js) | 框架 | 跨角色任务调度：优先级队列、资源锁、死锁检测、让位协调 |
| [`role.harvester.js`](../role.harvester.js) | 角色 | 采集 + 短途运输（兼容旧逻辑，目标数=0） |
| [`role.collector.js`](../role.collector.js) | 角色 | 纯采集，投放就近 Container 或掉落地上 |
| [`role.transporter.js`](../role.transporter.js) | 角色 | 全场搬运：取货 → 送货状态机，带资源锁与让位 |
| [`role.upgrader.js`](../role.upgrader.js) | 角色 | 升级控制器；短缺时远离 Spawn 暂停 |
| [`role.builder.js`](../role.builder.js) | 角色 | 建造工地，无工地时升级；短缺时暂停 |
| [`role.repairer.js`](../role.repairer.js) | 角色 | 修理受损建筑，无目标时升级 |
