# 01 · 项目概览

## 1.1 项目定位

本项目是一个面向 [Screeps](https://screeps.com/)（编程 MMO 游戏）的**单房间自动化管理 Bot**，使用模块化 JavaScript 编写。每 tick 由游戏引擎驱动 `module.exports.loop`，按角色调度 Creep 完成**采集、运输、建造、升级、修理与防御**，使房间自给自足运转。

## 1.2 核心特性

| 特性 | 说明 |
| --- | --- |
| **6 种角色状态机** | harvester、collector、transporter、upgrader、builder、repairer，职责分离 |
| **两级缓存降低 CPU** | Creep 名字缓存 + 矿点缓存，避免每 tick 全量 `_.filter(Game.creeps)` / `FIND_SOURCES` |
| **统一任务调度框架** | `task.scheduler.js` 提供优先级队列、资源锁、死锁检测、超时回收与让位协调 |
| **差异化身体部件策略** | `body.config.js` 按角色职责与能量阶段动态生成最优 body 组合 |
| **短缺优先机制** | 检测到角色数量不足时，Builder/Upgrader 自动暂停，把能量与 Spawn 槽位让给孵化 |
| **紧急模式重启** | 基地毁灭后检测 creep 总数过低，自动切换到 `emergencyRoleTargets`（以 harvester 为主，不依赖 Container）重启基地 |
| **动态 Spawn 发现** | 不再硬编码 `Spawn1`，每 tick 自动发现首个可用 spawn；检测到房间变更时自动清理失效状态与缓存 |
| **多重死锁/抖动修复** | 残余能量死锁、矿点满位顺延、Container↔Spawn 往返抖动、采集者让位等机制完善 |

## 1.3 技术栈

| 维度 | 选型 |
| --- | --- |
| 语言 | JavaScript（Screeps 运行时沙箱，Node.js 风格 `require` / `module.exports`） |
| 运行平台 | [Screeps 官方服务器](https://screeps.com/) |
| 构建工具 | **无** — 文件即模块，无需 npm/webpack |
| 持久化 | `Memory` 对象（Screeps 引擎自动序列化跨 tick 保存） |
| 瞬态状态 | `state.js` 单例（仅 tick 内有效，不写 `Memory`） |
| 调度模型 | 优先级队列 + 资源锁（sourceLocks / targetLocks） |

## 1.4 关键设计理念

1. **主循环极简**：`main.js` 只做编排，业务逻辑下沉到管理器与角色模块。
2. **瞬态状态与持久状态分离**：`state.js` 只在 tick 内有效，不写 `Memory`；持久状态由各模块按需写入 `Memory.creeps` / `Memory.tasks` / `Memory._transporterLocks`。
3. **缓存驱动**：角色调度、短缺检测、矿点选择均走缓存，O(1) 计数。
4. **职责单一**：每个 role 文件只负责一种 Creep 行为；manager 负责横切关注点（孵化、防御、任务调度）。
5. **采集与运输解耦**：collector 纯采集并投放就近 Container/地面，transporter 负责全场搬运，避免来回往返浪费 CPU。

## 1.5 角色一览

| 角色 | 目标数 | 职责 |
| --- | --- | --- |
| `harvester` | 0 | 采集 + 短途运输一体（兼容旧逻辑，当前主力为 collector+transporter） |
| `collector` | 6 | 纯采集，投放就近 Container 或掉落地上 |
| `transporter` | 5 | 全场搬运，连接采集端与消费端 |
| `upgrader` | 6 | 升级控制器；短缺时远离 Spawn 暂停 |
| `builder` | 5 | 建造工地；无工地时升级 |
| `repairer` | 1 | 修理受损建筑，含 Wall/Rampart |

> 默认配置见 [`config.js`](../config.js) 的 `roleTargets`，紧急模式见 `emergencyRoleTargets`。
