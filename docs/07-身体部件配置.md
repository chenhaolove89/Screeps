# 07 · 身体部件配置

[`body.config.js`](../body.config.js) 为不同角色提供差异化的部件生成策略。

---

## 7.1 设计原则

1. **移动效率优先**：确保平原地形无疲劳移动。
2. **能量利用最大化**：精确匹配可用能量。
3. **角色职责匹配**：专属化部件组合。
4. **生命周期适应**：支持不同发展阶段。

## 7.2 能量阶段（ENERGY_TIERS）

| 阶段 | 能量范围 | 名称 | 对应 RCL |
| --- | --- | --- | --- |
| `SURVIVAL` | 200-300 | 生存期 | 新房间启动 |
| `DEVELOPING` | 300-550 | 发展期 | RCL 2-3 |
| `GROWING` | 550-800 | 成长期 | RCL 4-5 |
| `MATURE` | 800-1300 | 成熟期 | RCL 6-7 |
| `PROSPERING` | 1300-3000 | 繁荣期 | RCL 8 |

## 7.3 角色配比策略

| 角色 | 配比策略 | 单元成本 | 上限单元数 | 设计理由 |
| --- | --- | --- | --- | --- |
| `harvester` | WORK:CARRY:MOVE = 2:1:2 | 350 | 4 | 平衡采集与移动，矿点-Spawn 高频往返 |
| `collector` | 最大化 WORK + 1 CARRY + 1 MOVE | 100 + 100×N | 8 WORK | 固定矿点工作，移动需求极低 |
| `transporter` | CARRY:MOVE = 1:1 | 100 | 8 单元 | 满载无疲劳，运输效率核心 |
| `upgrader` | WORK:CARRY:MOVE = 2:1:2 | 350 | 3 | 固定控制器附近，类似 harvester |
| `builder` | WORK:CARRY:MOVE = 2:1:2 | 350 | 4 | 多工地间移动 |
| `repairer` | WORK + 1 CARRY + N MOVE | 50 + 150×N | 5 WORK | 中等移动频率 |

### 低能量分支

为避免触发安全回退，部分角色在低能量时返回最小配置 `[WORK, CARRY, MOVE]`（成本 200）：

| 角色 | 触发条件 |
| --- | --- |
| `harvester` | `energy < 350` |
| `upgrader` | `energy < 350` |
| `builder` | `energy < 250` |
| `repairer` | `energy < 200` |

> `collector` 与 `transporter` 无低能量分支，因为其基础成本已足够低（100）。

---

## 7.4 主接口

### `getBody(energy, role)`

```js
bodyConfig.getBody(energy, role)  // 根据角色和能量返回 body 数组
```

执行流程：

1. **强制默认**：`config.bodyConfig.forceDefault === true` → 返回 `getDefaultBody(energy)`。
2. **策略选择**：按 role switch 到对应函数（`getHarvesterBody` 等），未知角色用 `getDefaultBody`。
3. **安全检查**：`calculateCost(body) > energy` → 打印警告，回退到 `getDefaultBody(energy)`。
4. 返回 body 数组。

### `calculateCost(body)`

按部件类型累加成本：

| 部件 | 成本 |
| --- | --- |
| MOVE | 50 |
| WORK | 100 |
| CARRY | 50 |
| ATTACK | 80 |
| RANGED_ATTACK | 150 |
| HEAL | 250 |
| TOUGH | 10 |
| CLAIM | 600 |

### `getEnergyTier(energy)`

遍历 `ENERGY_TIERS`，返回首个 `min <= energy <= max` 的阶段对象 `{tier, name}`，全部不匹配返回 `{tier:'PROSPERING', name:'繁荣期'}`。

---

## 7.5 默认配置（回退）

`getDefaultBody(energy)` 保持原有的 1:1:1 配比：

- 单元成本 200（WORK+CARRY+MOVE）。
- 单元数 `n = max(1, min(floor(energy/200), 8))`。
- 生成 `n×WORK + n×CARRY + n×MOVE`。

用于：
- 未知角色。
- 角色策略成本超能量时的安全回退。
- `forceDefault=true` 调试模式。

## 7.6 战斗角色预留

模块预留了战斗角色接口，**当前未启用**，为后续 PvP/扩张功能预留：

### COMBAT_ROLES

| 角色 | 标识 | 用途 |
| --- | --- | --- |
| `GUARD` | `guard` | 近战防御 |
| `RANGER` | `ranger` | 远程攻击 |
| `HEALER` | `healer` | 治疗 |
| `CLAIMER` | `claimer` | 占领房间 |

### COMBAT_BODY_TEMPLATES

| 角色 | low 配置 | high 配置 | 成本 |
| --- | --- | --- | --- |
| `guard` | `[TOUGH, ATTACK, ATTACK, MOVE, MOVE]` | `[TOUGH, TOUGH, ATTACK×3, MOVE×3]` | 270 / 310 |
| `ranger` | `[RANGED_ATTACK, MOVE]` | `[RANGED_ATTACK×2, MOVE×2]` | 200 / 400 |
| `healer` | `[HEAL, MOVE]` | `[HEAL×2, MOVE×2]` | 300 / 600 |
| `claimer` | `[CLAIM, MOVE]`（standard） | — | 650 |
