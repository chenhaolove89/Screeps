# 09 · 依赖关系

## 9.1 模块依赖图

```
main.js
├── role.harvester   → cache.sources
├── role.collector   → task.scheduler, cache.sources, state
├── role.transporter → task.scheduler
├── role.upgrader    → state, cache.sources, task.scheduler
├── role.builder     → state, cache.sources, task.scheduler
├── role.repairer    → cache.sources, task.scheduler
├── manager.spawn    → config, cache.creep, body.config, state
├── manager.tower    → state
├── task.scheduler   → (无内部依赖，操作 Memory.tasks)
├── state            → (无依赖)
└── cache.creep      → state

body.config          → config
cache.sources        → state
```

| `task.scheduler.js` | role.collector/transporter/upgrader/builder/repairer | 无（操作 Memory.tasks） | 自包含调度框架 |

## 9.4 角色间协作关系

```
collector ──投放能量──→ Container / 地面掉落
                            │
                      transporter 取货
                            │
                            ▼
            Spawn / Extension / Tower / Storage
                            │
            upgrader / builder / repairer 取能量
                            │
                            ▼
                    升级 / 建造 / 修理
```

### 协作链路说明

| 角色 | 产出 | 消费 |
| --- | --- | --- |
| **collector** | 纯采集，产出能量到 Container 或地面 | — |
| **transporter** | 全场搬运，连接采集端与消费端 | 从 Container/地面/Tombstone/Storage 取货 |
| **upgrader/builder/repairer** | 升级/建造/修理 | 从 Container/Spawn/Extension 取能量，能量枯竭时可回退直接采矿 |
| **harvester** | 兼容旧逻辑的采集+短途运输一体角色 | 当前目标数=0 |
| **manager.tower** | 防御 + 维修建筑 | 直接消耗 Tower 自身能量 |

### 让位优先级

```
collector 优先级最高（纯采集是能量链路源头）
    ↓ 连续 3 次无法到达 source
    ↓ requestYield
    ├── 优先驱离 transporter/upgrader/builder/repairer（YIELDABLE_ROLES）
    └── 其次调整其他 collector/harvester 站位
```

被请求让位的角色在 `run()` 入口调用 `checkYield`，处于让位期则移动到让位目标并跳过本 tick 正常逻辑。

## 9.5 数据流向

### 持久化数据（跨 tick）

| 存储位置 | 写入方 | 读取方 | 用途 |
| --- | --- | --- | --- |
| `Memory.creeps[name]` | Screeps 引擎（spawnCreep 时） + 各 role 模块（状态字段） | main（清理死亡 creep）、各 role 模块 | creep 状态机、任务绑定、健康记录 |
| `Memory.tasks` | task.scheduler | task.scheduler, role.transporter | 任务队列、资源锁、统计 |
| `Memory._transporterLocks` | role.transporter | role.transporter, main（清理失效锁） | 取货/送货锁 |
| `Memory._transporterLockTimestamps` | role.transporter | role.transporter | 锁超时检测 |

### 瞬态数据（tick 内）

| 存储位置 | 写入方 | 读取方 | 用途 |
| --- | --- | --- | --- |
| `state.creepShortage` | manager.spawn（checkShortage） | role.upgrader, role.builder | 短缺时暂停 |
| `state.allNames` / `byRole` | cache.creep | main, role.collector, manager.spawn | 调度循环、计数 |
| `state.sourceSlotCount` | cache.sources | cache.sources（hasFreeSlot） | 矿点空位 |
| `state.spawnName` / `spawnRoomName` | main | main, manager.spawn | Spawn 引用 |
