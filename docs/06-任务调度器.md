# 06 · 任务调度器

[`task.scheduler.js`](../task.scheduler.js) 是跨角色的统一任务调度框架，状态持久化在 `Memory.tasks`。

---

## 6.1 设计原则

1. 一个任务同一时刻只分配给一个 creep。
2. 资源节点分配采用「先占先得」。
3. 任务有超时机制，超时自动回收。
4. 所有状态持久化在 `Memory.tasks`。

## 6.2 常量

| 常量 | 值 | 说明 |
| --- | --- | --- |
| `TASK_TYPES` | `TRANSPORT/COLLECT/BUILD/REPAIR/UPGRADE` | 任务类型枚举 |
| `STATUS` | `PENDING/ASSIGNED/IN_PROGRESS/COMPLETED/FAILED/TIMED_OUT` | 任务状态枚举 |
| `PRIORITY` | `CRITICAL(0)/HIGH(1)/NORMAL(2)/LOW(3)` | 优先级（数字越小越高） |
| `TASK_TIMEOUT` | `300` ticks | 任务超时阈值 |
| `TASK_RETENTION` | `100` ticks | 已完成任务保留时长 |
| `MAX_ACTIVE_TASKS` | `50` | 最大活跃任务数 |
| `YIELD_TICKS` | `5` | 让位持续 tick 数 |
| `YIELD_DISTANCE` | `3` | 让位目标距 source 的最小距离 |
| `YIELDABLE_ROLES` | `['transporter','upgrader','builder','repairer']` | 可被驱离的非采集者 |

## 6.3 Memory.tasks 结构

```js
Memory.tasks = {
    active:        {},   // 活跃任务（pending/assigned/in_progress）
    completed:     {},   // 已完成任务（待 GC）
    sourceLocks:   {},   // source 资源锁（sourceId → creepName）
    targetLocks:   {},   // target 资源锁（targetId → [creepName, ...]）
    stats: {
        created:     0,
        completed:   0,
        failed:      0,
        timedOut:    0,
    },
};
```

## 6.4 核心 API

### 初始化与 GC

| 方法 | 说明 |
| --- | --- |
| `init()` | 确保 `Memory.tasks` 结构存在并执行 GC。应在 `main.js` 每 tick 开头调用 |
| `_gc()` | 清理过期已完成任务（`now - completedAt > TASK_RETENTION`） |

### 任务生命周期

| 方法 | 说明 |
| --- | --- |
| `createTask(type, priority, sourceId, targetId, resourceType, amount, maxRetries)` | 创建任务，含去重与容量检查，返回 taskId |
| `getNextTask(creep)` | 获取当前 creep 的下一个最优任务（按优先级，跳过已分配/锁冲突） |
| `assignTask(taskId, creepName)` | 分配任务给 creep，锁住 source |
| `updateTask(taskId, status, progressCurrent, progressTotal, error)` | 更新任务进度 |
| `completeTask(taskId)` | 完成任务，释放锁，归档，更新统计 |
| `failTask(taskId, error)` | 标记失败，未超 `maxRetries`（默认 3）则重置为 PENDING |

#### `createTask` 详解

1. 调用 `init()` 确保结构存在。
2. **容量检查**：活跃任务数 ≥ `MAX_ACTIVE_TASKS` 拒绝创建，返回 null。
3. **去重**：遍历 `active`，若已有相同 `type + sourceId + targetId` 且 `status=PENDING` 的任务，复用并返回其 taskId。
4. 生成 `taskId = 'task_' + Game.time + '_' + 随机6位`。
5. 写入 `active[taskId]`，含完整字段（priority/status/createdAt/assignedTo/retries/maxRetries 等）。
6. `stats.created++`。

#### `getNextTask(creep)` 详解

遍历 `active`，筛选 `status=PENDING` 且 `priority <= bestPriority` 的任务：
- 若 `sourceId` 被 `sourceLocks` 锁定且锁主非当前 creep → 跳过。
- 否则更新 `best` 与 `bestPriority`。
- 返回优先级最高（数字最小）的任务对象。

#### `failTask` 重试机制

- `retries++`，释放 source/target 锁。
- `retries < maxRetries` → 重置为 `PENDING`，清空 `assignedTo/assignedAt`。
- `retries >= maxRetries` → 归档为 `FAILED`，移入 `completed`，`stats.failed++`。

### 超时与死锁检测

| 方法 | 说明 |
| --- | --- |
| `checkTimeouts()` | 回收超时任务（`assigned/in_progress` 且 `now - assignedAt > TASK_TIMEOUT`） |
| `checkDeadlocks()` | 死锁检测：死 creep 持锁、停滞任务 |
| `releaseCreep(creepName)` | 释放 creep 持有的所有任务与锁 |

#### `checkDeadlocks` 三步检测

1. **sourceLocks**：持锁 creep 不存活 → 删除锁。
2. **targetLocks**：持锁 creep 不存活 → 从数组中过滤；空数组则删除键。兼容字符串旧格式。
3. **active 任务**：`assignedTo` 的 creep 不存活 → `retries++`，未超限重置为 PENDING，超限归档为 FAILED（`error='持有者死亡'`）。

#### `releaseCreep(creepName)`

释放指定 creep 持有的所有锁与任务：
- 遍历 `sourceLocks` / `targetLocks` 删除该 creep 的锁。
- 遍历 `active`，对该 creep 持有的任务执行重试逻辑（同 `checkDeadlocks` 第 3 步）。

> 调用时机：collector 卡住自恢复、transporter 停滞重置、creep 死亡。

## 6.5 让位协调机制

当 collector 无法到达 source（连续失败）时，调用 `requestYield` 请求 source 附近的其他 creep 让位。

### `requestYield(requester, source)` — 请求让位

```
查找 source.pos.findInRange(FIND_CREEPS, 1) 附近 1 格内的所有 creep
├─ 优先驱离非采集者（YIELDABLE_ROLES: transporter/upgrader/builder/repairer）
│   ├─ 找到 → _findYieldPosition(source) 找让位目标
│   │   ├─ 有 → 设置 c.memory._yieldUntil = Game.time + YIELD_TICKS
│   │   │        c.memory._yieldTarget = yieldPos
│   │   │        打印日志 → return true
│   │   └─ 无 → 继续
│   └─ 无 → 继续
└─ 其次调整其他 collector/harvester 的站位
    └─ 同上逻辑
```

### `checkYield(creep)` — 让位执行

各角色 `run()` 入口调用此方法：

1. 无 `_yieldUntil` → return false（正常执行）。
2. `Game.time > _yieldUntil` → 清理标记，return false。
3. 仍在让位期 → `moveTo(_yieldTarget, {stroke:'#ff4444', reusePath:5})`，return true（**跳过本 tick 正常逻辑**）。
4. 无 `_yieldTarget` → 清理标记，return false。

### `_findYieldPosition(source)` — 让位目标查找

在 source 外圈（`YIELD_DISTANCE` 到 `YIELD_DISTANCE+2` 格外）逐层查找空地：

1. 从 `r = YIELD_DISTANCE` 到 `YIELD_DISTANCE + 2` 逐层。
2. `room.lookAtArea(minY, minX, maxY, maxX, true)` 一次性获取区域。
3. 仅检查 Chebyshev 距离 = r 的外圈格。
4. 跳过 `terrain=wall` 与非 road/rampart 的结构。
5. `room.lookForAt(LOOK_CREEPS, x, y)` 检查无 creep 占据。
6. 返回首个可站立格 `{x, y, roomName}`，全无可站立格返回 null。
