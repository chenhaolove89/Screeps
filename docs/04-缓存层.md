# 04 · 缓存层

缓存层负责减少重复的 `Game.creeps` / `FIND_SOURCES` 扫描，是降低 CPU 占用的关键。

---

## 4.1 cache.creep.js — Creep 名字缓存

[`cache.creep.js`](../cache.creep.js) 维护 Creep 名字缓存，避免每 tick 多次 `_.filter(Game.creeps)`。仅在「初始化 / 孵化 / 死亡」三个节点刷新。

### 依赖

```js
var state = require('state');
```

### 核心 API

| 方法 | 说明 |
| --- | --- |
| `build()` | 从 `Game.creeps` 全量构建缓存（首 tick 或全局重置时调用）。遍历所有 creep 调用 `add(name)`，最后置 `state._cacheReady = true` |
| `add(name)` | 添加一只 creep 到缓存（孵化成功后调用）。push 到 `state.allNames` 与 `state.byRole[role]` |
| `remove(name)` | 从缓存移除一只死亡 creep。从 `state.allNames` 与对应 `state.byRole[role]` 中 splice，列表空时 delete 键 |
| `count(role)` | 获取某角色当前数量，O(1) 读 `state.byRole[role].length`，无则返回 0 |

### 刷新时机

| 时机 | 调用方法 | 调用方 |
| --- | --- | --- |
| 首 tick / 全局重置 | `build()` | `main.js` 主循环（`!state._cacheReady`） |
| 孵化成功 | `add(name)` | `manager.spawn.js` 的 `run()` |
| creep 死亡 | `remove(name)` | `main.js` 清理死亡 creep 内存循环 |

### 边界保护

- `add(name)` 检查 `creep` / `creep.memory` / `creep.memory.role` 任一不存在则直接 return。
- `remove(name)` 用 `indexOf` 查找后再 splice，未找到不报错。

---

## 4.2 cache.sources.js — 矿点缓存

[`cache.sources.js`](../cache.sources.js) 维护矿点缓存，矿点坐标房间固定，首 tick 初始化后直接读缓存。

### 依赖

```js
var state = require('state');
```

### 核心 API

| 方法 | 返回 | 说明 |
| --- | --- | --- |
| `getSorted(creep)` | `{id, dist}[]` | 按 Chebyshev 距离排序矿点（最近优先） |
| `hasFreeSlot(source)` | `boolean` | 矿点周围是否还有空闲可站立格（每 tick 缓存） |
| `getFreeSlotCount(source)` | `number` | 获取矿点周围空闲格数量（0-8），用 `lookAtArea` 检测墙/creep/墙结构 |
| `getSpawnDistance(source)` | `number` | 获取矿点到 Spawn 的距离（缓存） |
| `getSourcesBySpawnDistance(room)` | `Source[]` | 返回按到 Spawn 距离排序的矿点列表（近→远） |
| `harvestNearest(creep)` | `boolean` | 尝试采集最近可用矿点，满位时自动顺延到下一矿点；所有矿点不可用时兜底走向最近矿点 |

### `getSorted(creep)` — 距离排序

遍历 `state.sourceIds`，对每个矿点计算 `max(|creep.x - data.x|, |creep.y - data.y|)`（Chebyshev 距离），升序排序后返回 `[{id, dist}]`。

### `getFreeSlotCount(source)` — 空闲格检测

每 tick 第一次调用时计算并缓存到 `state.sourceSlotCount[sourceId]`：

1. `room.lookAtArea(sy-1, sx-1, sy+1, sx+1, true)` 一次性获取矿点周围 3×3 区域。
2. 标记被 `terrain=wall` / `creep` / `STRUCTURE_WALL` 占据的格为 `blocked`。
3. 遍历 8 个相邻格（跳过中心），统计未 blocked 的格数。

> 注意：road/rampart/container 等**非墙结构不阻挡站立**，不计入 blocked。

### `getSpawnDistance(source)` — Spawn 距离缓存

- 命中 `state.sourceSpawnDist[sid]` 直接返回。
- 未命中时 `room.find(FIND_STRUCTURES, {filter: spawn})[0]` 取首个 spawn，计算 Chebyshev 距离并写入缓存。
- 无 spawn 时返回 `Infinity`。

### `getSourcesBySpawnDistance(room)` — 容错重建

1. 从 `state.sourceIds` 缓存逐个 `Game.getObjectById` 取 source。
2. **缓存全部失效**（迁移新房间未及时重建）→ 直接 `room.find(FIND_SOURCES)` 重建 `sourceIds` / `sourceData` / `sourceSpawnDist`，并打印 `[SourceCache] 缓存失效，已从 room.find 重建`。
3. 按 `getSpawnDistance` 升序排序后返回。

### `harvestNearest(creep)` — 满位顺延采集

```
遍历 getSorted(creep) 返回的矿点列表（近→远）:
  ├─ creep.harvest(source)
  │   ├─ OK → return true
  │   ├─ ERR_NOT_IN_RANGE
  │   │   ├─ hasFreeSlot(source)=false（饱和）→ continue 顺延下一矿点
  │   │   └─ 有空位 → moveTo(source) → return true
  │   └─ 其他错误 → 尝试下一矿点
└─ 所有矿点不可用 → 兜底 moveTo(sorted[0]) → return true
```

> 此函数是 harvester / upgrader / builder / repairer 在能量枯竭时的统一兜底采集入口。
